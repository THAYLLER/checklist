# checklist
Sobre o teste de desenvolvimento:

- Você deve fazer um aplicativo simples de checklist (como Todoist) e publica-lo no Github ou Bitbucket.

Nesse sisteminha eu devo poder: 
- Cadastrar uma tarefa. 
- Editar essa tarefa.
- Apagar essa tarefa. 
- Marcar uma tarefa pendente como feita.
- Marcar uma tarefa feita como pendente novamente.

Requisitos:
O sistema deve ser feito com:
- Frontend em Angular e Bootstrap ou Material
- Backend RESTFUL em Node.js usando Express
- Deve-se usar o Sequelize ORM no Backend
- Deve-se gerar migrations e seeders do Sequelize
- Banco de dados: MYSQL

- Não é necessário login e autenticação.
- Não é necessário escrever testes automatizados.
- Não é necessário utilizar Typescript no backend (mas seria legal).

Sobre a realização:
Por se tratar de um teste não queremos nada elaborado ou muito trabalhoso.
O objetivo desse teste é entender melhor como é seu nível de conhecimento, escrita e capricho, então quanto mais você conseguir nos transparecer isso melhor.
Com base nesses itens iremos elaborar uma proposta de trabalho para você.
